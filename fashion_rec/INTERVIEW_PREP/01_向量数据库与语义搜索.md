# 向量数据库与语义搜索 - 技术面试准备

## 1. 核心概念解释

### Q: 什么是向量数据库？为什么需要它？

**答案：**
向量数据库是专门用于存储和检索高维向量数据的数据库系统。在我们的项目中，我们使用向量数据库来：

1. **存储图像嵌入向量**：使用CLIP模型将服装图像转换为512维的向量表示
2. **语义相似度搜索**：通过计算向量之间的余弦相似度，找到视觉上相似的服装
3. **高效检索**：相比传统的关键词搜索，向量搜索能理解图像的语义内容

**技术细节：**
- 使用 `sentence-transformers` 库的 `clip-ViT-B-32` 模型
- 生成512维的向量嵌入
- 使用Supabase的pgvector扩展存储向量

### Q: 为什么选择Supabase pgvector而不是ChromaDB或Pinecone？

**答案：**

**选择Supabase pgvector的原因：**

1. **数据持久化**：pgvector基于PostgreSQL，数据持久化更可靠，不会因为服务重启丢失数据
2. **统一数据管理**：用户数据、向量数据、元数据都在同一个数据库中，简化架构
3. **SQL查询能力**：可以结合传统SQL查询和向量搜索，实现复杂的过滤条件
4. **成本效益**：相比Pinecone等托管服务，Supabase更经济实惠
5. **事务支持**：支持ACID事务，保证数据一致性

### Q: CLIP模型的工作原理是什么？

**答案：**

**CLIP (Contrastive Language-Image Pre-training) 模型：**

1. **双编码器架构**：
   - 图像编码器：Vision Transformer (ViT) 将图像转换为向量
   - 文本编码器：Transformer将文本转换为向量
   - 两者映射到同一个向量空间

2. **对比学习训练**：
   - 训练时，匹配的图像-文本对在向量空间中距离更近
   - 不匹配的对距离更远
   - 通过大量图像-文本对训练，学习语义表示

3. **在我们的应用中的使用**：
   ```python
   # 使用CLIP模型生成图像嵌入
   from sentence_transformers import SentenceTransformer
   
   model = SentenceTransformer('clip-ViT-B-32')
   embedding = model.encode(image)  # 生成512维向量
   ```

4. **优势**：
   - 零样本学习：无需针对特定任务训练
   - 语义理解：理解图像的语义内容，不仅仅是像素相似
   - 跨模态：图像和文本在同一个向量空间

## 2. 实现细节

### Q: 如何优化向量搜索的性能？

**答案：**

**性能优化策略：**

1. **索引优化**：
   ```sql
   -- 使用IVFFlat索引加速向量搜索
   CREATE INDEX ON wardrobe_items 
   USING ivfflat (embedding vector_cosine_ops)
   WITH (lists = 100);
   ```
   - `lists`参数控制索引精度和速度的平衡
   - 更大的lists值 = 更精确但更慢
   - 更小的lists值 = 更快但可能不够精确

2. **查询优化**：
   ```python
   # 限制搜索范围，只搜索用户自己的衣橱
   query = supabase.table('wardrobe_items')\
       .select('*')\
       .eq('user_id', user_id)\
       .limit(limit)
   ```

3. **缓存策略**：
   - 缓存常用查询的向量嵌入
   - 使用模型缓存避免重复加载模型

4. **批量处理**：
   ```python
   # 批量生成嵌入向量，提高效率
   embeddings = model.encode(images, batch_size=32)
   ```

### Q: 如何处理向量搜索的准确性？

**答案：**

**准确性保证：**

1. **相似度阈值**：
   ```python
   # 设置最小相似度阈值，过滤掉不相关的结果
   min_similarity = 0.7  # 余弦相似度阈值
   results = [r for r in results if r['similarity'] >= min_similarity]
   ```

2. **混合搜索策略**：
   - 向量搜索：找到视觉相似的物品
   - 元数据过滤：结合颜色、类型、风格等结构化属性
   - 最终结果：综合向量相似度和元数据匹配度

3. **多模态增强**：
   - 使用Qwen-VL提取结构化属性（颜色、类型、风格）
   - 向量搜索找到视觉相似物品
   - 两者结合提高准确性

## 4. 常见问题与解决方案

### Q: 如果向量搜索返回的结果不准确怎么办？

**答案：**

**解决方案：**

1. **调整相似度阈值**：根据实际效果调整最小相似度
2. **增强元数据**：使用Qwen-VL提取更丰富的属性
3. **混合搜索**：结合向量搜索和关键词搜索
4. **用户反馈**：收集用户反馈，持续优化

### Q: 向量数据库的存储成本如何？

**答案：**

**成本分析：**

1. **向量大小**：512维 × 4字节 = 2KB per item
2. **元数据**：JSON格式，通常 < 1KB per item
3. **总存储**：每个物品约3KB
4. **1000件衣服**：约3MB
5. **Supabase免费额度**：500MB，足够小规模使用

```python
# 准备一个完整的向量搜索实现示例
def search_similar_items(
    user_id: str,
    query_image: Image,
    limit: int = 10,
    min_similarity: float = 0.7
) -> List[Dict]:
    """搜索相似物品的完整实现"""
    # 1. 生成查询向量
    model = get_clip_model()
    query_embedding = model.encode(query_image)
    
    # 2. 执行向量搜索
    results = supabase.rpc(
        'match_wardrobe_items',
        {
            'query_embedding': query_embedding,
            'match_threshold': min_similarity,
            'match_count': limit,
            'user_id': user_id
        }
    ).execute()
    
    # 3. 返回结果
    return results.data
```
